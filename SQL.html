1. Primary Key vs Unique Key

Primary Key vs Unique Key
Primary KeyUnique Key
Uniquely identifies each rowEnsures column values are unique
Only one per tableMultiple allowed
Does not allow NULLAllows one NULL
Clustered index by defaultNon-clustered index
-- Example
CREATE TABLE Students (
  ID INT PRIMARY KEY,
  Email VARCHAR(100) UNIQUE
);

2. Clustered vs Non-Clustered Index

Clustered vs Non-Clustered Index
Clustered IndexNon-Clustered Index
Sorts physical dataSeparate pointer structure
Only one per tableMultiple allowed
Fast for range queriesFast for exact matches
CREATE TABLE Employees (
  EmployeeID INT PRIMARY KEY, -- Clustered
  Name VARCHAR(100)
);

CREATE NONCLUSTERED INDEX idx_Name ON Employees(Name);
